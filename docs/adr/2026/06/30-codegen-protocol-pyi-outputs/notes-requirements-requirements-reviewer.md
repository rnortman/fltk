## requirements-1

**Section / quote:** Change 2, path 1 — "In this case the generator already knows the protocol module
path -- it just wrote the file -- so the `.pyi` is an automatic byproduct. No extra flags or parameters
are needed."

**What's wrong:** The doc earlier defines `protocol_module` as "a dotted Python import path (e.g.,
`fltk.fegen.fltk_cst_protocol`) that is interpolated verbatim into the stub's import line." The claim
that the generator "knows the protocol module path" *because* "it just wrote the file" conflates the
filesystem path where the protocol `.py` lands with the dotted import path the `.pyi`'s
`import {protocol_module} as _proto` line requires. Writing a file at some path does not by itself yield
a dotted import path — that depends on the protocol module's identity/package location. In practice the
co-generation opt-in will have to be told the protocol's module identity (just as the Python `generate`
subcommand already takes `base_name` / `cst_module_name`), and the import path is derivable from that.

**Why:** This is the refiner's own elaboration, not the user's words. The user directive says only "If
it's being generated by the rust generator, then we automatically know it's there and we can generate the
.pyi." The directive is about *not having to re-supply* the import path (i.e. not double-specifying via
`--protocol-module`), not a claim that the opt-in needs no module-identity input at all.

**Consequence:** Small. A careful designer has enough context (the Python path takes a module name;
`generate_pyi` takes a dotted path) to connect the dots and will give the Rust protocol opt-in a
module-identity parameter. But the near-absolute "No extra flags or parameters are needed" + the
file-write-implies-import-path framing could be misread as licensing a bare boolean opt-in that carries
no module name, which would then have nothing to feed the `.pyi` import line.

**Suggested fix:** Tighten to match the user's actual intent — e.g. "When the Rust generator co-generates
the protocol, the protocol module's import path is already supplied (as part of enabling protocol
output), so the `.pyi` reuses it automatically; the caller does not separately pass `--protocol-module`."
Drop the "it just wrote the file" causal phrasing.
