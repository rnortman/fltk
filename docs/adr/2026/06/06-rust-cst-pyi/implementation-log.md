## Increment 2 — `.pyi` emitter: `generate_pyi` method on `RustCstGenerator` (commit e5d1656)

- `fltk/fegen/gsm2tree_rs.py:109-227`: added `RustCstGenerator.generate_pyi(protocol_module: str) -> str` and private `_pyi_annotation_for_model_types`. - `fltk/fegen/genparser.py:264-335`: added `--protocol-module` and `--pyi-output` options to `gen-rust-cst`. Generates `.pyi` text before opening any file (mirrors partial-file-avoidance pattern). Without `--protocol-module`, no `.pyi` emitted (backward compatible). Removed `TODO(rust-cst-pyi)` comment.
- `TODO.md`: removed the `rust-cst-pyi` entry.
- `tests/test_gsm2tree_rs.py:713-920`: 28 new unit tests across `TestGeneratePyiHeader`, `TestGeneratePyiClasses`, `TestGeneratePyiPerLabelAccessors`, `TestGeneratePyiModuleLevelClassAttrs`, `TestGeneratePyiClassLabelSetMatchesRs`. All pure string assertions; no Rust toolchain required.

## Increment 3 — pyright self-check and stub-vs-protocol conformance tests (§2.2, §4) (commit d57a034)

- `fltk/fegen/gsm2tree_rs.py:146-154`: Fixed `Label` emission: changed from `Label: typing.ClassVar[type[_proto.Class.Label]]` to `Label = _proto.Class.Label` (type alias assignment). ClassVar caused pyright `reportRedeclaration` in self-check and `"Label" is not defined as a ClassVar in protocol` in conformance check.
- `fltk/fegen/gsm2tree_rs.py:172`: Fixed `extend_children` parameter: `other: _proto.{class_name}` instead of `other: {class_name}`. Using the stub-local class caused contravariance failures in conformance checks.
- `fltk/fegen/gsm2tree_rs.py:192-198`: Removed module-level `<Class>: type[<Class>]` attrs — they caused `reportRedeclaration` in the self-check. CstModule conformance works from class definitions alone once Label and extend_children are fixed.
- `tests/test_gsm2tree_rs.py`: Updated 4 existing assertions to match the corrected generator output (`test_labelled_rule_has_label_alias`, `test_zero_label_rule_has_no_label`, `test_extend_children_present`, `TestGeneratePyiModuleLevelClassAttrs` rewritten).
- `tests/test_gsm2tree_rs.py`: Added pyright harness (`pyright_available` fixture, `_run_pyright_in_tmpdir`, `_write_pyi_tmpdir`, fixture helpers).
- `tests/test_gsm2tree_rs.py`: Added `TestGeneratePyiSelfCheck` (2 tests: fegen + PoC pyi self-check zero errors).
- `tests/test_gsm2tree_rs.py`: Added `TestGeneratePyiConformance` (2 tests: whole-module no-cast zero errors, per-class no-cast zero errors for all 14 fegen classes).
- All 108 tests pass; `uv run pyright` 0 errors repo-wide.
- Deviation: design §2.1 specified `Label: typing.ClassVar[type[_proto.<Class>.Label]]`; shipped `Label = _proto.<Class>.Label` and no module-level `<Class>: type[<Class>]` attrs. Both design-specified forms cause pyright errors (self-check `reportRedeclaration`; conformance `"Label" is not defined as a ClassVar`). The corrected forms satisfy both self-check and conformance (verified by the new tests).
