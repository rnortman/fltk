## 3. Phase 4 Integration Targets

### 3.1 `check-no-pyo3` — two stanzas missing

The controlling design §5 item 5 specifies "no-pyo3 assertion for `fltk-parser-core` **and for the fixture parser crate built `--no-default-features`**." The fixture parser crate (`tests/rust_parser_fixture`) has default features empty (Cargo.toml line 5-6: `[features]` with `python` and `extension-module` but no `default`), so `cargo tree --manifest-path tests/rust_parser_fixture/Cargo.toml --edges normal,build` with no `--features` flag is the python-off assertion. Phase 3 §2.7 explicitly deferred this: "The cargo tree assertion stays Phase 4."

`tests/rust_cst_fegen` has default features `extension-module` (python-on), but `--no-default-features` is the python-off path and is already tested in `cargo-test-no-python` and `cargo-clippy-no-python`. Whether `check-no-pyo3` should also cover it is a judgment call the design leaves to Phase 4 consolidation.

Current `check-no-pyo3` positive control: uses `fltk-cst-core` presence as the health-check. For `rust_parser_fixture`, the positive control must be `fltk-parser-core` presence (which is a direct dependency; Cargo.toml:8 `fltk-parser-core = { path = "../../crates/fltk-parser-core" }`).

### 3.2 Self-hosting test — one new test class

The Rust-parser path: `fegen_rust_cst.Parser(text, False).apply__parse_grammar(0)` returns `ApplyResult | None`. `r.result` is the canonical `fegen_rust_cst.Grammar` handle. `fltk2gsm.Cst2Gsm(text)` (not `TerminalSource.terminals` — `Cst2Gsm.__init__` takes the raw `str`, `fltk2gsm.py:11`) can then `visit_grammar(r.result)` because `Cst2Gsm.visit_grammar` calls `grammar.children_rule()`, `.child_name()`, `.child_value()`, etc. — the same accessor surface the AC8 test already exercises via the Python IIR parser path. The success check is `r.pos == len(text)` (Python `len()` returns codepoint count for str, matching Rust codepoint positions).

The design calls for extending `tests/test_phase4_fegen_rust_backend.py` with a new test class, not adding a new plumbing path.

### 3.3 ADR README

`docs/adr/2026/06/10-rust-parser-codegen/README.md` does not exist. The controlling design §3.1 references it for documenting the regex-crate-subset restriction and §6 specifies it as a Phase 4 deliverable recording the §2 "direct Rust emission" decision.
