# Request: fegen-cst-rs-single-source

Style: concise, precise, no padding, no preamble. Audience: smart LLM/human.

**Type of work:** drift-elimination refactor (build/file layout); small.

**Background.** Two files are byte-for-byte identical (confirmed: same md5, 5080 lines) but committed independently:
- `src/cst_fegen.rs` — generated by `python -m fltk.fegen.genparser gen-rust-cst` (Makefile `gen-rust-cst`); declared `mod cst_fegen;` in `src/lib.rs:5`, registered into `fltk._native.fegen_cst`.
- `tests/rust_cst_fegen/src/cst.rs` — declared `mod cst;` in `tests/rust_cst_fegen/src/lib.rs:11`; built as the separate `fegen_rust_cst` cdylib (Makefile `build-fegen-rust-cst`); consumed by `tests/test_phase4_fegen_rust_backend.py`, `tests/test_clean_protocol_consumer_api.py`, `tests/test_cross_backend_label_equality.py` via `importorskip("fegen_rust_cst")`.

Only the first has a regeneration command. Nothing regenerates the second; after a grammar change they silently diverge and the test crate compiles stale generated code. This is the drift foot-gun to eliminate.

**Fix shape (chosen).** Make `src/cst_fegen.rs` the single source of truth and have the test crate include it rather than hold a copy. Replace `tests/rust_cst_fegen/src/cst.rs` contents with a Rust `include!` of the canonical file: `include!("../../../src/cst_fegen.rs");` (path resolves relative to the including file). `tests/rust_cst_fegen/src/lib.rs` keeps `mod cst;`. This is the validated-cleanest option: no Cargo workspace, no symlink (avoids Windows `core.symlinks=false` / git-blob issues), no separate copy step needed from a clean checkout.

**Load-bearing constraints.**
- The three crates (`Cargo.toml`, `tests/rust_cst_fegen/Cargo.toml`, `tests/rust_cst_fixture/Cargo.toml`) are independent cdylibs, NOT a Cargo workspace. Do not introduce a workspace.
- `src/cst_fegen.rs` starts with top-level `use pyo3::...;` items; `include!`-ing it inside `mod cst { ... }` (via `mod cst;` loading the one-line file) places those `use` items at module scope — valid Rust. Verify it compiles.
- `make fix` / generated-code-formatting flow still applies to `src/cst_fegen.rs`.
- Do NOT touch `tests/rust_cst_fixture/` (a different fixture, the `phase4_roundtrip_cst` crate) — out of scope.
- Bazel has no rules for `tests/rust_cst_fegen` (Rust unsupported under Bazel today) — no Bazel impact.

**Non-goals.** No grammar regeneration. No change to how `src/cst_fegen.rs` itself is generated. No workspace. No `.gitignore` of generated files (keep `src/cst_fegen.rs` committed).

**Verification.** `git diff --no-index src/cst_fegen.rs <expanded include>` conceptually equal; `cd tests/rust_cst_fegen && uv run --group dev maturin develop` builds; the three consuming test files pass (`uv run pytest tests/test_phase4_fegen_rust_backend.py tests/test_clean_protocol_consumer_api.py tests/test_cross_backend_label_equality.py`). Removing/editing `src/cst_fegen.rs` would now necessarily change the test crate's compiled output (no second copy to drift). `TODO.md` entry and the `TODO(fegen-cst-rs-single-source)` comment at the top of `tests/rust_cst_fegen/src/cst.rs` removed.

**Exploration:** `exploration.md` in this dir.
