scope-1: tests/fltk/fegen/test_name_validation.py — tests 1–4 use validate_no_underscore_only_names directly instead of plumbing.generate_parser as the design specified. If plumbing.generate_parser had a code path that bypassed classify_trivia_rules (thus skipping the validator), tests 1–4 would all pass yet the end-to-end pipeline would still raise IndentationError. The design's stated purpose for tests 1–4 is to lock the "before: IndentationError, after: ValueError" contract at the pipeline level, not just the validator level. No existing test calls plumbing.generate_parser with a rule named _ or __ to confirm the plumbing integration. Consequence: the plumbing integration is untested; a future refactor that introduces a bypass path would not be caught by the test suite. Suggested fix: add (or convert) at least tests 1–2 to call plumbing.generate_parser(parse_grammar("_ := val:/[a-z]+/ ;")) as the design prescribed.

scope-2: tests/fltk/fegen/test_name_validation.py (design test 9 — capture_trivia=True pipeline run) — the design requires a test that runs the full plumbing pipeline with capture_trivia=True to confirm the auto-added _trivia rule (label 'content') passes through the validator without raising. test_classify_trivia_rules_with_trivia_passes satisfies only the label_foo regression and a direct classify_trivia_rules call; no test exercises a capture_trivia=True plumbing run. Consequence: if the validator were mistakenly tightened to reject _trivia (or the auto-added 'content' label), no test would catch it. The full-suite green check (test 10) provides implicit coverage only if plumbing self-hosting tests exercise capture_trivia; that is incidental, not the explicit lock the design intended. Suggested fix: add a test that calls plumbing.generate_parser with capture_trivia=True on a grammar that has no explicit _trivia rule, confirming no ValueError is raised.

Both omissions are single-file, small additions (each a ~5-line test). Aggregate scope is well within respond-mode patching — no escalation warranted.
