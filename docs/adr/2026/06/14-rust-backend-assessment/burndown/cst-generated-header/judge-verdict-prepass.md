This is a deliberate, context-correct choice, not drift: `_rust_str_lit` (`gsm2parser_rs.py:31`) escapes for a Rust *string literal* (`\` → `\\`, `"` → `\"`), whereas the CST header interpolates the name inside a *backtick doc-comment span*. Reusing `_rust_str_lit` here would have emitted `\\` sequences into a doc comment. The divergence is the right call; no finding owed.
