a `source_name` ctor param threaded from `gen_rust_cst`.

No findings.

Notes:

  a parse/render/per-request hot path). O(1) string formatting; no loops, no I/O.
- `_escape_source_name()` does two `str.replace` passes over a short path string,
  once per generation. Negligible.
- No per-rule or per-node work added; the new line is appended to `parts` before
  the existing per-rule loop, outside it.
- `source_name` stored as a single str on the instance; no unbounded growth, no
  retained references, no listener/cleanup concerns.
- No new blocking work on any startup or per-request path. No missed concurrency
  (single-threaded codegen, nothing to parallelize here).
