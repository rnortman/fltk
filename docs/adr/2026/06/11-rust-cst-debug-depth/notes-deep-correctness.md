## What was verified (logic traces, not just reading)

- **Steal soundness (`strong_count == 1`)**: `Shared` wraps `Arc<RwLock<T>>`; grep over `crates/` + `src/` confirms zero `Arc::downgrade`/`Weak` usage (registry weakness is Python `WeakValueDictionary` on PyO3 handles, which hold *strong* `Shared`). Observing count 1 while holding a handle ⇒ sole owner; no concurrent clone possible (cloning requires another handle). Check-then-steal is sound single-threaded; the concurrent-dropper race nests at most one node-Drop per racing dropper (each is itself iterative), per design §edge-cases — bounded by dropper count, not tree depth.
- **Lock discipline**: `mem::take(&mut shared.write().children)` guard is statement-scoped, released before `worklist.extend` and before `shared` drops at arm end; node `Drop` takes no locks (operates on `&mut self` fields). No nested guards, no same-thread re-entry.
- **Termination/aliasing**: per-node steal happens at most once (requires count 1, stealer owns the handle). Duplicate handles to one child in a worklist (DAG, or same child pushed twice): first drain sees count>1 → decrement only; last drain sees count 1 → steal. Traced correct.
- **Drop early-return invariant**: stolen nodes drop childless → `is_empty()` early return; span-only union members (`Num`, `Name`, `Trivia`, …) have no `Drop` impl at all, and their drop glue (span + empty Vec) is non-recursive. Never-child roots (`Items`, `Grammar`) correctly have `Drop` + `into_drop_item` but no `DropWorklistItem` variant — only `into_drop_item` constructs variants, so a root variant would be dead code.
- **`needs_drop_item` gating** (`gsm2tree_rs.py:594`): emitted iff `child_classes` non-empty (called by own `Drop`) or class ∈ child-union (called by `drain_into` arm). Both call-site sets covered; no uncalled emission (dead_code) and no missing emission (compile error). Confirmed in generated output: `IdentifierChild`/`TriviaChild` get span-only `into_drop_item` (union members), `ItemsChild` gets full mapping, `DropWorklistItem` = {Identifier, Trivia} exactly.
- **Union closure**: every class name in any `drain_into` arm's stolen-children type is itself a union member by construction (`_child_class_union` unions `_child_variants_for_rule` over all rules), so every `into_drop_item` referenced exists.
- **Debug non-recursion**: node `Debug` reads own fields only; child-enum derived `Debug` → `Shared::Debug` (one read lock) → node manual `Debug` terminates at depth 1. `format_args!` temporary lives through the builder-chain statement (valid).
